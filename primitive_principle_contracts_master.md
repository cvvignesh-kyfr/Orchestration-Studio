# Core Philosophy

## Primitive

A primitive is:

> A stable behavioral control dimension used to shape runtime conversational behavior.

Primitives are:
- Atomic
- Continuous
- Composable
- Observable
- Independently tunable
- Runtime resolvable

Primitives are NOT:
- Personas
- Narratives
- Flows
- Prompts
- Scripts
- Features

Primitives are low-level behavioral controls.

---

# Principle

A principle is:

> A runtime behavioral interpretation of one or more primitives.

Principles define:
- How primitives manifest behaviorally
- What the runtime should optimize for
- How conflicts should resolve
- What the user experiences

Principles are semantic.
Primitives are mechanical.

---

# Relationship Model

```text
Primitive
→ Controls measurable behavioral dimension

Principle
→ Defines behavioral interpretation and orchestration intent
```

Example:

```text
Primitive:
  warmth = 0.82

Principle:
  "The assistant should feel emotionally supportive and psychologically safe."
```

# Canonical Runtime Rules

## Rule 1 — Primitive Semantics Are Immutable

The meaning of a primitive can NEVER change.

Example:

```text
warmth
```

Must ALWAYS mean:

```text
Emotional softness, friendliness, and psychological comfort.
```

It can NEVER later mean:
- verbosity
- empathy
- enthusiasm
- humor

Those are separate primitives.
